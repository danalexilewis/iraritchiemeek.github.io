Friday 19 December 2014

Why should I use version Control? There are many benifits to using a version control system such such as Git, here are just a few to convince you.

  • It allows you to save many versions of the same file as you progress. This means if you mess up you can easily go back to a previous version instead of having to start over.
  • You can experiment on multiple versions of your code, and then easily look back at them and choose the best version.
  • If a bug/issue is introduced there are commands you can use to find out where that bug was introduced and then rollback to that version.
  • You always have multiple backups of your work.
  • If you're working in teams you can work on the same file at the same time and see who made the changes and when.
  • It provides you with a full history of your code and keeps track of changes you have made that you can easily look back on.

GitHub could be thought of as an online version of Git although there are some differences between the two. Storing your code on GitHub allows others to look at your code, copy your code, suggest changes and give you feedback.
